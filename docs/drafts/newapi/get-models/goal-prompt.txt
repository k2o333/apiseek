## 核心契约（必须实现）

### 协议与模块
- 入口：newapi_monitor.py（扩展 CLI/编排/锁）；新模块 newapi_models.py（Token 写 + models 风险面）。
- **鉴权：** monitor 持有 Session / user_id / 一次性恢复；models 只注入 list/create/update/get-secret；/v1/models **永不** session 重登。
- **不要** 新增无价值的 --once；无 models flag 时保持现网 **默认单轮 groups**。
- 新增 CLI（互斥）：--models-preflight | --models-bootstrap | --models-refresh。
- 组键：norm_group = str.strip；managed 名：newapi-monitor:g: + norm_group。
- 用户 Token：**永不删除、永不 PUT**。

### P0 行为
1. **Hydration：** list_tokens_all 分页完整后，先对相关 Token get_token_secret，内存 hydrated 后再算 Missing；secret 临时失败 → coverage_unknown → **fail closed 不 create**。
2. **inventory-suitable：** 绑组 + secret 可读 + 启用 + 未过期 + model_limits_enabled=false + allow_ips 适合 inventory；**仅**此可阻止 create managed；受限 Token 的 models **不得**标为全量成功快照。
3. **分页 fail-closed：** has_more=True 必须续页；total 不足不 complete；incomplete 禁止 create；max_pages + 无进展保护。
4. **reconcile：** create 带 group；未知 POST 结果禁止盲重 POST；managed 禁用/过期/错绑 → **仅 managed PUT 修复**（探针确认载荷）；返回 hydrated tokens_after。
5. **门禁：** preflight 零写；需 seed Token；bootstrap 强制 preflight；refresh 无 bootstrap → exit≠0 零写。
6. **ModelsStore：** 仅 models_latest.json + models_events.jsonl；schema 对齐 Sub2API（null vs []、失败保旧、逐组 checkpoint、full meta 含 **skipped**、bootstrap_completed_at）。
7. **T-new：** MONITOR_MODELS_INCREMENTAL_ENABLE 默认 0；仅真正 added 且已 bootstrap；**ensure 仅 refresh_set** 非全量 G；T-new 部分失败 groups 仍 exit 0。
8. **退出码：** bootstrap/refresh 时 failed>0 或 skipped>0 → ≠0；last_full_result={target,ok,failed,skipped}；target=ok+failed+skipped。
9. **锁：** 默认 groups 非阻塞；bootstrap/refresh 有界等待 + 进程内有限重试。
10. **systemd：** newapi-models-daily@：OnCalendar=*-*-* 00:00:00 Asia/Shanghai + RandomizedDelaySec=300 + TimeoutStartSec=600 + ExecStart --models-refresh；install **默认不 enable** daily。
11. **安全：** 落盘/日志无完整 key/session/password。

### 非目标
- Sub2API 站 models 路径；botcf 强制 live bootstrap（须 preflight；实现可共用代码）
- keys_index / 默认 keys_secrets / 自动删用户 Token / 00:30 backup timer
- SQLite / sites.yaml / provider registry
- 改变现网空 groups contract（空 object 失败、不覆盖旧快照）
- 默认把缺快照并入每 5 分钟全量 models

## Acceptance criteria
1. CLI 暴露 --models-preflight / --models-bootstrap / --models-refresh；preflight 永不 create/写 models；**默认无 models flag 的 groups 单轮**在无 bootstrap 或增量关时：零 create、零 models 请求。
2. hydration → inventory-suitable → Missing；list_tokens 分页 fail-closed；managed 名与 reconcile/修复符合 requirements；tokens_after 用于 models；用户 Token 不改删。
3. 仅新增 models_latest + models_events；null/[]/失败保旧/checkpoint/full meta+skipped/bootstrap 标志正确。
4. 触发器/auth 分域/T-new 范围/exit 码/锁/日更 unit 符合 requirements。
5. 真实单测覆盖 P0 路径 + tests/test_newapi_monitor.py 与全仓相关测试仍绿；无密钥泄漏。
