# AIAPIBANK（aiapibank.com）技术栈与接口分析

## 1. 结论速览

- `https://www.aiapibank.com/` **不是** QuantumNous/new-api 的前端范式。
- 该站点前端技术栈与 **Wei-Shaw/sub2api** 完全一致，判断为 sub2api 的二次部署/换皮站点。
- 站点版本号：`0.1.160`。
- 用户账号示例：`干死小仙女`（id=95，邮箱 zyc0926@gmail.com）。密码: bblswuji1

## 2. 前端技术栈对比

### 2.1 QuantumNous/new-api
- 框架：React 19
- 构建工具：Rsbuild（Rspack）
- 模板：
  - `web/default`：React 19 + Base UI + Tailwind CSS + TanStack Router
  - `web/classic`：React 18 + Semi Design + React Router 6
- 包管理：Bun

### 2.2 aiapibank.com（实测）
- 框架：Vue 3
- 构建工具：Vite（hash 资源文件名，如 `index-Dkv4qXvN.js`）
- UI：TailwindCSS
- 路由：Vue Router
- 状态管理：Pinia
- 页面根节点：`<div id="app" data-v-app="">`

### 2.3 Wei-Shaw/sub2api
- 框架：Vue 3.4+
- 构建工具：Vite 5+
- UI：TailwindCSS
- 路由：Vue Router
- 状态管理：Pinia
- 包管理：pnpm
- 前端目录：`frontend/`，构建后输出到 `../backend/internal/web/dist`

**结论**：aiapibank.com 与 sub2api 完全吻合，与 new-api 不匹配。

## 3. 已确认的关键 API 端点

| 端点 | 方法 | 说明 | 认证 |
|---|---|---|---|
| `/api/v1/settings/public` | GET | 站点公开配置 | 不需要 |
| `/setup/status` | GET | 站点初始化状态 | 不需要 |
| `/api/v1/groups/available` | GET | 所有可用分组及倍率 | 需要 Bearer Token |
| `/api/v1/groups/rates` | GET | 分组倍率 | 需要 Bearer Token |
| `/api/v1/channels/available` | GET | 可用渠道 | 需要 Bearer Token |
| `/api/v1/keys` | GET | 用户 API Key 列表（含分组信息） | 需要 Bearer Token |

### 3.1 认证方式
- 从浏览器 `localStorage` 读取 `auth_token`。
- Header 格式：`Authorization: Bearer <JWT>`。
- 错误示例：
  - 无 Header：`{"code":"UNAUTHORIZED","message":"Authorization header is required"}`
  - 格式错误：`{"code":"INVALID_AUTH_HEADER","message":"Authorization header format must be 'Bearer {token}'"}`

### 3.2 Token 有效期与刷新
- `localStorage` 中同时存在：
  - `auth_token`
  - `token_expires_at`（时间戳）
  - `refresh_token`
- 长期自动化任务需要处理 Token 刷新，或定期从浏览器重新拷贝更新。

### 3.3 登录接口（已实测可用）

| 端点 | 方法 | Content-Type | 请求体 | 说明 |
|---|---|---|---|---|
| `/api/v1/auth/login` | POST | `application/json` | `{"email":"...","password":"..."}` | 账号密码登录 |

**实测验证**：
- API 直接调用：返回 200 及 `access_token` / `refresh_token`。
- 浏览器表单登录：在 `/login` 页面填写邮箱、密码，点击「登录」后成功跳转至 `/dashboard`，`localStorage` 写入 `auth_token`。

返回示例：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "rt_..."
  }
}
```

后续请求使用 `Authorization: Bearer <access_token>` 即可。

#### curl 示例
```bash
curl -s -X POST https://www.aiapibank.com/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"your_email","password":"your_password"}'
```

#### Python 登录示例
```python
import requests

BASE_URL = "https://www.aiapibank.com"

def login(email: str, password: str) -> dict:
    r = requests.post(
        f"{BASE_URL}/api/v1/auth/login",
        headers={"Content-Type": "application/json"},
        json={"email": email, "password": password},
        timeout=30,
    )
    r.raise_for_status()
    return r.json()["data"]

# data = login("your_email", "your_password")
# token = data["access_token"]
```

> **安全提醒**：不要把真实密码写进文档或代码仓库。生产环境请用环境变量或密钥管理工具注入。

## 6. Linux 服务器定时监控方案

### 6.1 目标
每 1 分钟调用 `/api/v1/groups/available`，记录所有分组名称、倍率、状态。

### 6.2 环境准备
```bash
# 安装依赖
pip3 install requests

# 方式一：直接设置已有 token
export AIAPIBANK_TOKEN="eyJhbGciOiJIUzI1NiIs..."

# 方式二：设置账号密码，让脚本自己登录获取 token
export AIAPIBANK_EMAIL="your_email"
export AIAPIBANK_PASSWORD="your_password"
```

### 6.3 监控脚本 `/opt/aiapibank/check_groups.py`
```python
#!/usr/bin/env python3
import os
import json
import requests
import datetime
from pathlib import Path

BASE_URL = "https://www.aiapibank.com"
LOG_FILE = Path("/var/log/aiapibank_groups.jsonl")


def login(email: str, password: str) -> str:
    r = requests.post(
        f"{BASE_URL}/api/v1/auth/login",
        headers={"Content-Type": "application/json"},
        json={"email": email, "password": password},
        timeout=30,
    )
    r.raise_for_status()
    return r.json()["data"]["access_token"]


def get_token() -> str:
    # 优先使用环境变量里的 token
    token = os.environ.get("AIAPIBANK_TOKEN")
    if token:
        return token

    # 否则用账号密码登录
    email = os.environ.get("AIAPIBANK_EMAIL")
    password = os.environ.get("AIAPIBANK_PASSWORD")
    if not email or not password:
        raise RuntimeError("请设置 AIAPIBANK_TOKEN 或 AIAPIBANK_EMAIL+AIAPIBANK_PASSWORD")
    return login(email, password)


def check():
    token = get_token()
    r = requests.get(
        f"{BASE_URL}/api/v1/groups/available",
        headers={"Authorization": f"Bearer {token}"},
        timeout=30,
    )
    r.raise_for_status()
    payload = r.json()
    groups = payload.get("data", [])

    record = {
        "ts": datetime.datetime.now(datetime.timezone.utc).isoformat(),
        "count": len(groups),
        "groups": [
            {
                "id": g["id"],
                "name": g["name"],
                "rate_multiplier": g["rate_multiplier"],
                "status": g["status"],
            }
            for g in groups
        ],
    }

    LOG_FILE.parent.mkdir(parents=True, exist_ok=True)
    with LOG_FILE.open("a", encoding="utf-8") as f:
        f.write(json.dumps(record, ensure_ascii=False) + "\n")

    print(f"[{record['ts']}] recorded {record['count']} groups")


if __name__ == "__main__":
    check()
```

### 6.4 部署方式一：systemd timer（推荐）

把账号密码放到独立文件 `/opt/aiapibank/.env`，并限制权限：
```bash
echo 'AIAPIBANK_EMAIL=your_email' > /opt/aiapibank/.env
echo 'AIAPIBANK_PASSWORD=your_password' >> /opt/aiapibank/.env
chmod 600 /opt/aiapibank/.env
```

`~/.config/systemd/user/aiapibank-check.service`
```ini
[Unit]
Description=Check AIAPIBANK groups

[Service]
Type=oneshot
EnvironmentFile=/opt/aiapibank/.env
ExecStart=/usr/bin/python3 /opt/aiapibank/check_groups.py
```

`~/.config/systemd/user/aiapibank-check.timer`
```ini
[Unit]
Description=Run AIAPIBANK group check every minute

[Timer]
OnBootSec=1min
OnUnitActiveSec=1min
AccuracySec=1s

[Install]
WantedBy=timers.target
```

启用：
```bash
systemctl --user daemon-reload
systemctl --user enable --now aiapibank-check.timer
systemctl --user list-timers
```

### 6.5 部署方式二：cron
```bash
* * * * * export AIAPIBANK_EMAIL="your_email"; export AIAPIBANK_PASSWORD="your_password"; /usr/bin/python3 /opt/aiapibank/check_groups.py >> /var/log/aiapibank_cron.log 2>&1
```

### 6.6 部署方式三：常驻循环脚本
适合需要实时告警的场景：
```python
import time
while True:
    check()
    time.sleep(60)
```
配合 systemd 或 supervisor 保活。

## 7. 注意事项与后续建议

1. **Token 安全**：不要把 token 直接写死在脚本里提交到仓库，优先用环境变量或 secret manager。
2. **Token 刷新**：JWT 会过期，长期无人维护的方案建议：
   - 用 `refresh_token` 实现自动刷新；或
   - 写一个小工具定期从浏览器导出 token 到服务器。
3. **频率控制**：1 分钟一次通常安全，若遇 Cloudflare 限流可降到 5 分钟，并补充 `User-Agent` 等浏览器头。
4. **数据过滤**：如只关心 `status == "active"` 的分组，在脚本中过滤即可。
5. **告警扩展**：可在 `check()` 后对比上次结果，若分组/倍率变化则发送钉钉/企业微信/邮件通知。
6. **进一步分析**：
   - `/api/v1/channels/available` 当前返回空数组，可能需管理员权限或特定参数。
   - `/api/v1/keys` 可获取用户自己的 API Key 及其所属分组详情。
