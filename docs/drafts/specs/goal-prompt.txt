## 工作方式（强制）

1. 用 **不同 subagent** 分工（可并行则并行）：
   - **governance/manifest**：Phase 0 文档与 manifest 结构；
   - **checker/tools**：`tools/contract_check.py` + artifact loader；
   - **fixtures/schemas**：某 contract 的 schema/vectors（与实现接线可并行起草，但 status 变更必须串行）；
   - **tests/contract**：门禁与 vectors 重放；
   - **review**：对照 execution-plan §6–§11 出口条件。
2. 改 **同一文件**（尤其 `manifest.yaml`、governance、`contract_check.py`）必须 **串行**，避免冲突。
3. subagent 若不自动返回，**每 10 分钟**查一次任务输出，直到完成或 blocked。
4. 自己交付全部结果：不把手工步骤甩给用户；不中途问「要不要继续」。
5. **TDD / 证据驱动**：
   - 先有 schema/fixture/vector 或失败测试，再补齐实现或 checker；
   - vectors **必须** 调用 shipped 正式 writer/reader/adapter（禁止在测试里重写一份生产算法只测自己）；
   - HTTP **只在传输边界 mock**；不把在线 curl 成功当 contract evidence。
6. 完成前执行本提示词 **Verification plan**，观察成立后再 `update_goal(completed: true)`。
7. 用 todo 跟踪 Phase 与出口条件；每完成一 Phase 在 `{SCRATCH}/phase-N-exit.md` 写清证据路径与命令输出摘要。

## 硬门禁（违反即不合格）

1. **Phase 0 完成前，禁止** 将 manifest 中任何 contract 或 bundle 改为 `status: frozen`。
2. **禁止** 在 `docs/drafts/` 建立第二套「正式」契约权威；frozen 后 artifact **不得** 引用 `docs/drafts/`。
3. **禁止** artifact/fixture 用 glob 指向 `data/` 或真实 `sites/*.env`。
4. **禁止** 用「PR 描述口头 N/A」跳过 gate；N/A 必须写入 manifest + 治理允许 + `reviewed_in`。
5. **禁止** 未满足 dependency / capability 时冻结消费者 contract。
6. **禁止** 扩大默认远端 mutation；BotCF models mutation **保持不可启用**。
7. **禁止** 改变生产 timer + oneshot 模型；deployment 变更须保持 models timer **默认 disabled**。
8. **禁止** 失败路径覆盖最后成功业务快照（groups/models/invite）。
9. **禁止** secret 进入普通 stdout/日志/业务 latest/events/fixtures（canary 扫描）。
10. 单项冻结 PR 顺序（若本 goal 触及 status）：先证据与测试在 `draft` 下通过 checker → **最后** 才改 `frozen` → CI 再验。

## 核心设计约束（摘自 plan，实现时不得违反）

### Manifest / artifacts

- `required_artifacts` 每个 id 必须在对应 `artifact.yaml` 中存在。
- 路径：仓库根相对；解析后不得逃出仓库。
- `kind` 来自 checker 固定枚举；新增 kind 先改工具与测试。
- frozen 后不得引用 drafts / 真实 data / secret env（本 goal 先实现检查逻辑）。

### Fixtures / vectors

- positive 全 accept；negative 全 reject；negative 用稳定 `error_kind`（文件名或 sidecar），不绑定第三方完整报错串。
- vector 单 case 目录：`case.yaml` + inputs + expected；runner registry 调正式代码。

### Secret scan

- canary password/JWT/session/API Key/proxy userinfo；
- 扫描 stdout/stderr/日志/latest/events；
- 拒绝高风险结构；允许 `key_id`/`error_kind`；禁止粗糙 `key` 正则误报。

### 实现改动原则

- **先兼容读，再升级写**；涉及现网文件格式时先 reader/vectors，再动 writer。
- 契约边界单一：storage / CLI / provider / config / mutation / deployment 不互相夹带未归属语义。
- 不把 `docs/drafts/specs` 当第二状态源；状态查看用 checker `list` 或直接读 manifest。

## Verification plan

在仓库根执行；输出进 `{SCRATCH}/`。

1. **gating — 依赖与 checker**
   ```bash
   .venv/bin/python -c "import jsonschema, yaml; print(jsonschema.__version__)"
   .venv/bin/python tools/contract_check.py list
   .venv/bin/python tools/contract_check.py validate
   ```
   → `{SCRATCH}/checker-validate.log`

2. **gating — contract + 全量测试**
   ```bash
   .venv/bin/python -m unittest discover -s tests/contract -t . -v
   .venv/bin/python -m unittest discover -s tests -t . -v
   ```
   → `{SCRATCH}/unittest-contract.log`、`unittest-all.log`

3. **gating — 负例门禁**（至少 2 类）
   - 临时移走某 required artifact 或改坏 schema → validate/tests 非零；
   - canary 故意写入假 latest → secret 测试失败。  
   恢复工作树后记录 → `{SCRATCH}/negative-gates.log`

4. **gating — 试点一致性**
   - positive fixtures 全过、negative 全拒；
   - hash/transition vectors 与 `monitor_storage`（或试点选定的正式模块）结果一致；
   - 对同一 vectors **跑 2 次** 结果一致 → `{SCRATCH}/pilot-replay.log`

5. **gating — 状态审计**
   ```bash
   # 确认无意外 frozen（本 goal 默认）
   .venv/bin/python tools/contract_check.py list --status frozen
   ```
   → `{SCRATCH}/status-audit.log`（期望：仅原有状态或仍无 frozen，与 goal 范围一致）

6. **evidence**
   - `{SCRATCH}/hypothesis.md`：plan 章节 → 仓库文件映射；
   - `{SCRATCH}/manifest-diff-summary.md`：manifest/governance 变更摘要；
   - secret-scan 与 review 笔记。

7. **deployment 命令**（仅当本 goal 改动了 unit/installer 时）
   ```bash
   systemd-analyze verify ./*.service ./*.timer
   ```

全部 gating 观察成立后，才可标记 goal 完成。

## Implementation approach

1. **Hypothesis**：通读 execution-plan §1–§12，输出 `{SCRATCH}/hypothesis.md`（Phase 0/1 文件树、试点证据矩阵、风险）。
2. **Phase 0 串行**：governance → 正式 specs 边界 → manifest 结构扩展 → planned 条目；**不改 status 为 frozen**。

4. **试点 2B**：从现有 `monitor_storage.py` + `tests/test_newapi_monitor.py` **提炼** schema/vectors（schema 权威来自正式 storage 契约，不是“当前输出原样 dump”）；绑定真实 writer/reader。

6. 缺口写入 `{SCRATCH}/ready-to-freeze.md` / 后续 goal 边界。
